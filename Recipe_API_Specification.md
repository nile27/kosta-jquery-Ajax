# 📋 조리식품 레시피 DB API 연동 과제

## 1. 프로젝트 개요
* **서비스명**: 조리식품의 레시피 DB
* **제공기관**: 식품의약품안전처
* **사용 스택**: Jquery, Ajax, HTML5, CSS3
* **데이터 설명**: 식품의 레시피를 이미지화하고 만드는 방법, 열량, 성분에 대한 속성 정보를 제공합니다.

---

## 2. API 호출 정보 (Request)

### 기본 요청 주소
* **URL 구조**: `http://openapi.foodsafetykorea.go.kr/api/인증키/서비스명/요청파일타입/요청시작위치/요청종료위치`
* **샘플 링크**: [조리식품 레시피 샘플](http://openapi.foodsafetykorea.go.kr/api/sample/COOKRCP01/xml/1/5)

### 요청 인자 상세
| 번호 | 변수명 | 타입 | 필수여부 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | keyId | STRING | 필수 | Open API에서 발급받은 인증키 |
| 2 | serviceId | STRING | 필수 | 요청 대상 서비스명 (예: COOKRCP01) |
| 3 | dataType | STRING | 필수 | 요청 파일 타입 (xml, json) |
| 4 | startIdx | STRING | 필수 | 요청 시작 위치 (정수 입력) |
| 5 | endIdx | STRING | 필수 | 요청 종료 위치 (정수 입력) |
| 6 | RCP_NM | STRING | 선택 | 추가 파라미터: 메뉴명 |
| 7 | RCP_PARTS_DTLS | STRING | 선택 | 추가 파라미터: 재료 정보 |
| 8 | CHNG_DT | STRING | 선택 | 변경일자 기준 출력 (YYYYMMDD) |
| 9 | RCP_PAT2 | STRING | 선택 | 추가 파라미터: 요리 종류 (반찬, 국, 후식 등) |

---

## 3. 출력 항목 상세 (Response)

| 항목명 | 설명 | 항목명 | 설명 |
| :--- | :--- | :--- | :--- |
| **RCP_SEQ** | 일련번호 | **INFO_PRO** | 단백질 |
| **RCP_NM** | 메뉴명 | **INFO_FAT** | 지방 |
| **RCP_WAY2** | 조리방법 | **INFO_NA** | 나트륨 |
| **RCP_PAT2** | 요리 종류 | **HASH_TAG** | 해쉬태그 |
| **INFO_WGT** | 중량(1인분) | **ATT_FILE_NO_MAIN** | 이미지경로(소) |
| **INFO_ENG** | 열량 | **ATT_FILE_NO_MK** | 이미지경로(대) |
| **INFO_CAR** | 탄수화물 | **RCP_NM** | 만드는 법 정보 (1~20단계) |

---

## 4. 웹 사이트 기능 요구사항
1. **디자인 컨셉**: 녹색 계열의 트렌디한 컬러 사용 및 깔끔한 UI/UX 구현.
2. **오늘의 메뉴 추천**: 메인 화면에서 랜덤으로 메뉴를 노출하는 기능.
3. **무한 스크롤**: 레시피 목록을 무한 스크롤 방식으로 로딩.
4. **상세 페이지**: 클릭 시 재료, 조리 방법, 이미지, 영양소 등 상세 정보 표시.
5. **검색 기능**: 메뉴 이름 혹은 재료 키워드로 검색 가능 기능.
6. **코드 주석**: 각 기능의 역할에 대한 상세 주석 작성.

---

## 5. 추가 레퍼런스 (이미지 첨삭 내용)
* **동적 파라미터 구성**: 이미지 가이드에 따라, 추가 요청 인자가 있을 경우 URL 끝에 `/변수명=값&변수명2=값2` 형태로 결합하여 호출해야 합니다.
* **필터링 활용**: `RCP_NM`(메뉴명)이나 `RCP_PARTS_DTLS`(재료명) 변수를 활용하여 사용자가 입력한 검색어에 맞는 데이터를 API 단계에서 필터링하여 가져올 수 있습니다.
* **업데이트 주기**: 본 데이터의 최종 수정일은 2021-08-03이며, 호출 한도는 기본 1,000건입니다. (인증키 권한에 따라 상이할 수 있음)
